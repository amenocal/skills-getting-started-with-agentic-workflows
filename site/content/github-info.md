## Latest GitHub Updates

- **GitHub Copilot app is now available in technical preview** (May 14, 2026): A GitHub-native desktop experience that lets you steer Copilot cloud agent through focused coding sessions—start from GitHub context, validate changes, and ship pull requests without leaving the app. — _GitHub Changelog_

- **Copilot cloud agent supports auto model selection** (May 14, 2026): When you choose "Auto" in the model picker, the cloud agent picks the best available model dynamically; you get a 10 % discount on the model multiplier and avoid weekly rate limits. — _GitHub Changelog_

- **Start Copilot cloud agent tasks via the REST API** (May 13, 2026): A new public-preview Agent Tasks REST API lets Copilot Business and Enterprise users trigger cloud agent coding sessions programmatically, making it easy to wire Copilot into custom automations. — _GitHub Changelog_

- **Copilot CLI agent and unified sessions view in JetBrains IDEs** (May 13, 2026): JetBrains plugin users can now run the Copilot CLI agent inside their IDE and track all active Copilot sessions in a single unified view. — _GitHub Changelog_

- **Team-level Copilot usage metrics now available via API** (May 14, 2026): The Copilot usage metrics API adds a `user-teams` report that maps licensed users to their teams, making it easier to analyse adoption by team without building custom joins. — _GitHub Changelog_

- **GitHub Actions: Upcoming image migrations** (May 14, 2026): Arm64 runner images are now maintained by GitHub; `macos-latest` and Windows 2025 / Visual Studio 2026 migrations begin June 15—check your workflows now to avoid surprises. — _GitHub Changelog_

- **Copilot code review: Comment experience improvements** (May 12, 2026): Code review comments now include severity levels and are grouped by file to reduce noise, making it faster to triage Copilot suggestions in pull requests. — _GitHub Changelog_

- **GitHub Weekly Comment Sync workflow** (Awesome Copilot Workflows): A community agentic workflow that runs on a schedule to find stale code comments or README snippets, syncs them automatically, and opens a draft PR when changes are needed—great for keeping docs in sync with code. — _Awesome Copilot Workflows_

- **GitHub Relevance Check slash command** (Awesome Copilot Workflows): A community workflow triggered by `/relevance-check` that uses Copilot to evaluate whether an issue or pull request is still relevant to the project, helping teams keep their backlog clean. — _Awesome Copilot Workflows_
