## Latest GitHub Updates

- **GPT-5.3-Codex is now the base model for Copilot Business and Enterprise**: GitHub upgraded the default Copilot model to GPT-5.3-Codex, meaning Business and Enterprise users get improved code understanding and generation without any configuration change. — GitHub Changelog
- **Copilot Memory supports user preferences for Pro and Pro+ users**: Copilot can now remember and apply your personal coding preferences across sessions, reducing repetitive instruction and making suggestions feel more tailored over time. — GitHub Changelog
- **GitHub App installation tokens: Per-request override header**: Developers building GitHub Apps can now override the installation token scope on a per-request basis, giving finer-grained control over API access in multi-tenant tooling. — GitHub Changelog
- **GitHub Actions: Upcoming image migrations**: Several GitHub-hosted runner images are being migrated; developers should review their workflows now to avoid breaking changes when the rollout completes. — GitHub Changelog
- **Team-level Copilot usage metrics available via API**: Teams can now query Copilot usage at the team level through the REST API, making it easier to measure and report on productivity gains across squads. — GitHub Changelog
- **GitHub Copilot app now in technical preview**: A standalone Copilot app is available in technical preview, giving developers a dedicated AI coding interface outside the IDE for exploration and quick tasks. — GitHub Changelog
- **GitHub Copilot individual plans — flex allotments and new Max plan**: GitHub introduced flex allotments for Pro and Pro+ plans alongside a new Max plan, giving individual developers more flexible and predictable AI usage options. — GitHub Blog
- **From latency to instant: Modernizing GitHub Issues navigation performance**: The engineering team shares how they dramatically reduced Issues page load times — useful reading for developers building or maintaining web performance-sensitive applications. — GitHub Blog
- **Daily Issues Report (community workflow)**: This agentic workflow runs on a schedule to generate a daily summary of open issues and recent repository activity, posted directly as a GitHub issue for async team visibility. — Awesome Copilot Workflows
- **OSPO Organization Health Report (community workflow)**: A weekly agentic workflow that surfaces stale issues and PRs, merge time analysis, and contributor leaderboards across an organization, giving open source maintainers actionable health metrics. — Awesome Copilot Workflows
- **OSS Release Compliance Checker (community workflow)**: Triggered by an issue comment, this workflow analyzes a repository against open source release requirements and posts a detailed compliance report, helping teams ship releases with confidence. — Awesome Copilot Workflows
